## Sprint 1: Construcción del MVP

### Sprint Retrospective

Al cierre del Sprint 1, el equipo realizó la retrospectiva para evaluar cómo se desarrolló el trabajo, qué prácticas ayudaron al avance del MVP y qué aspectos debían fortalecerse para el siguiente sprint.

### Lo que salió bien

- Se logró construir una base funcional del MVP y presentarla de forma clara en la Sprint Review.
- El uso de Jira permitió visualizar mejor el avance, las tareas pendientes y la distribución del trabajo.
- La documentación web comenzó a consolidarse como un punto de referencia útil para el seguimiento del proyecto.

### Lo que debemos mejorar

- Tener una comunicación más constante entre todos los integrantes del equipo.
- Reforzar especialmente la comunicación en el backend para evitar desalineaciones técnicas.
- Mantener la documentación más activa durante el sprint y no solo al cierre.
- Discutir con mayor frecuencia decisiones, bloqueos y avances entre áreas.

### Acuerdos de mejora

- Mantener reuniones frecuentes por Discord en las tardes o noches para revisar el estado del proyecto.
- Utilizar esos espacios para discutir avances, bloqueos y decisiones con el comité de mantenimiento.
- Dar seguimiento más cercano a los temas de backend y a sus integraciones con frontend.
- Actualizar la documentación de manera continua conforme avance el sprint.
- Fortalecer la coordinación del equipo para que los cambios importantes se comuniquen con mayor rapidez.

### Enfoque para el siguiente sprint

Los acuerdos de esta retrospectiva se tomarán como lineamientos de trabajo para el siguiente sprint, con prioridad en comunicación, documentación activa y mejor sincronización técnica del equipo.
