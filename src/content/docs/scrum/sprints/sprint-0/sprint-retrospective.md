## Sprint 0: Fase de Cimientos y Prototipado

### Sprint Retrospective

Al cierre del Sprint 0, el equipo realizó la retrospectiva para evaluar cómo se desarrolló el trabajo, qué prácticas ayudaron al avance de los cimientos del proyecto y qué aspectos debían fortalecerse para el siguiente sprint.

### Lo que salió bien

- El equipo se organizó y completó todas las tareas a pesar de no contar con herramientas formales al inicio.
- Los prototipos de Figma dieron una dirección visual clara y alinearon al equipo sobre el producto a construir.
- Se sentaron las bases técnicas necesarias para el MVP: backend, frontend, CI/CD y modelo de datos.

### Lo que debemos mejorar

- Configurar Jira y las herramientas de gestión desde el inicio del sprint, no hacia el final.
- Establecer canales formales de comunicación y reuniones periódicas desde el día 1.
- Definir y respetar las ceremonias Scrum como parte fija de cada sprint.
- Documentar el proceso de manera continua durante el sprint, en lugar de hacerlo al cierre.

### Acuerdos de mejora

- Configurar el tablero de Jira con todas las tareas antes de iniciar el Sprint 1.
- Establecer un canal principal de comunicación en Discord para el día a día del equipo.
- Programar reuniones diarias breves para dar visibilidad al avance y detectar bloqueos a tiempo.
- Mantener la documentación activa y actualizada conforme avance el sprint.

### Enfoque para el siguiente sprint

Los acuerdos de esta retrospectiva se tomarán como lineamientos de trabajo para el siguiente sprint, con prioridad en formalizar el marco Scrum completo e iniciar la construcción del MVP con historias de usuario funcionales.
