## Estructura del Equipo Scrum

El equipo de EmprendeX está organizado siguiendo los roles definidos por el marco de trabajo Scrum.

<Aside type="tip" title="Equipo Scrum">
  En Scrum, el equipo es auto-organizado y multifuncional. Todos son responsables del éxito del proyecto, aunque cada rol tiene responsabilidades específicas.
</Aside>

---

## Roles

### Product Owner (PO)

El Product Owner es responsable de maximizar el valor del producto y gestionar el Product Backlog.

| Campo | Detalle |
|-------|---------|
| **Nombre** | (Nombre del PO) |
| **Responsabilidades** | Priorizar el backlog, validar historias en la Review, representar al cliente |

### Scrum Master (SM)

El Scrum Master facilita el proceso Scrum y elimina impedimentos para el equipo.

| Campo | Detalle |
|-------|---------|
| **Nombre** | (Nombre del SM) |
| **Responsabilidades** | Facilitar ceremonias, eliminar bloqueos, asegurar el proceso Scrum |

### Equipo de Desarrollo

El equipo de desarrollo es responsable de entregar incrementos funcionales al final de cada Sprint.

| Nombre | Rol Principal | Área de Enfoque |
|--------|---------------|-----------------|
| (Nombre) | Developer | Frontend / React Native |
| (Nombre) | Developer | Backend / API |
| (Nombre) | Developer | Full Stack / QA |

## Canales de Comunicación

| Canal | Propósito | Frecuencia |
|-------|-----------|------------|
| (WhatsApp / Discord / etc.) | Comunicación diaria del equipo | Diario |
| Reunión de Daily Scrum | Sincronización del equipo | Cada día |
| GitHub | Gestión de código y PRs | Continuo |
| (Jira / Trello / Notion) | Gestión del backlog y sprints | Cada Sprint |

## Acuerdos del Equipo

Estos son los acuerdos que el equipo estableció al inicio del proyecto:

- **Horario de Daily:** (hora) todos los días vía (plataforma).
- **Disponibilidad esperada:** (X) horas por semana por miembro.
- **Convención de commits:** `feat:`, `fix:`, `docs:`, `chore:` (Conventional Commits).
- **Convención de ramas:** `feature/US-XX-descripcion`, `fix/descripcion`.
- **PRs:** Requieren al menos 1 aprobación antes de hacer merge a `main`.
